Ollama、open-webui安装教程

Jan 27, 2025
阅读时长: 2 分钟

什么是Ollama？

Ollama是一个用于在本地运行大语言模型的开源工具。

准备工作：安装英伟达驱动和CUDA

这里因为我们使用的是Ubuntu系统，在初始安装的时候要把允许第三方驱动勾上，系统安装完成后，通过以下命令安装驱动工具

apt update && apt install ubuntu-drivers-common

安装完成后，运行以下命令自动安装服务器显卡驱动

ubuntu-drivers --gpgpu install

然后安装Cuda，这里我偷懒直接APT安装了，你也可以从英伟达官网下载最新版本

apt install nvidia-cuda-toolkit

这个包比较大，请耐心等待，安装完成后我们准备工作就完成了，可以通过 nvidia-smi 查看显卡信息

第一步：安装Ollama

通过官网提供的一键脚本安装Ollama，网址没有被墙大陆机器也可以直接运行

curl -fsSL https://ollama.com/install.sh | sh

安装完成后，可以使用 systemctl status ollama 查看一下Ollama的运行状态，正常是active(running)，没问题我们进入下一步安装open-webui

第二步：使用docker-compose安装open-webui

在Ubuntu的初始安装界面可以选择安装docker，如果你当时没有安装，可以通过以下命令一键安装

wget -qO- https://get.docker.com | sh --mirror Aliyun

请注意：通过Ubuntu初始设置安装的docker是通过snap安装的，如果需要修改daemon.json，文件位置在 /var/snap/docker/current/config/daemon.json（而不是/etc/docker/damon.json），修改完成后，需要运行 snap restart docker 来应用变化

新建一个open-webui的文件夹，复制compose文件如下：

services:
  open-webui:
    image: ghcr.io/open-webui/open-webui:main
    container_name: open-webui
    restart: unless-stopped
    network_mode: host
    volumes:
      - open-webui:/app/backend/data
    environment:
      OLLAMA_BASE_URL: "http://127.0.0.1:11434"
      ENABLE_OPENAI_API: False
volumes:
  open-webui:

如果是大陆的机器，请自行寻找ghcr的镜像

运行 docker compose up -d 拉起容器，此时可以通过 http://IP:8080 访问open-webui

第三步：运行大语言模型

根据你机器的配置规格运行一个大语言模型，这里以deepseek为例

ollama run --verbose deepseek-r1:8b

如果成功调用GPU，那么此时运行 ollama ps 会显示PROCESSOR为100% GPU，也可以通过 nvidia-smi 查看GPU负载

至此部署完成~
